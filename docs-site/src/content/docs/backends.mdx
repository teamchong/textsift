---
title: Backends
description: Three execution engines behind the same public API.
---

textsift ships up to three interchangeable backends. The public API (`PrivacyFilter.create`, `redact`, `detect`) is identical across all of them вҖ” the backend is an implementation detail picked at construction.

| Backend | Where | Compute |
|---|---|---|
| `webgpu` | `textsift-core` and `textsift` | Custom WGSL kernels (int4 matmul, banded attention, sparse MoE). Requires `shader-f16`. |
| `wasm` | `textsift-core` and `textsift` | Custom Zig + SIMD128 вҶ’ WASM. Multi-thread when COOP/COEP headers set. |
| `auto` (transformers.js fallback) | `textsift` only | ORT-Web with WebGPU EP. Used by the umbrella's auto path on browsers without WebGPU and without SIMD-WASM. |

## Compatibility

| | Chromium 147 | Firefox 129 | Safari 19 | Node 22 |
|---|:-:|:-:|:-:|:-:|
| `webgpu` | вң… | вҡ пёҸ `shader-f16` preview | вҡ пёҸ `shader-f16` preview | вқҢ |
| `wasm` | вң… | вң… | вң… | вң… |
| `auto` (`textsift` only, falls back to transformers.js) | вң… | вң… if `dom.webgpu.enabled=true` | вң… | вҡ пёҸ via ORT-Node |

`backend: "wasm"` is the only currently-working CPU path for `model_q4f16.onnx`. transformers.js with `device: "wasm"` fails session creation because ORT-Web's WASM bundle has no implementation for the `GatherBlockQuantized` and `MatMulNBits` ONNX contrib ops the int4 export uses.

## Output parity

All backends produce **byte-identical spans** on the same input (`tests/browser/smoke.spec.ts` + `webgpu-smoke.spec.ts`). Logit magnitudes drift by up to ~0.2 RMS due to fp16 rounding differences accumulating across 8 transformer layers, but argmax (and hence Viterbi + span decode) is preserved.

## Switching backends at runtime

A `PrivacyFilter` instance is bound to one backend for its lifetime. To switch, dispose and recreate:

```ts
const wasm = await PrivacyFilter.create({ backend: "wasm" });
await wasm.redact(text);
wasm.dispose();

const gpu = await PrivacyFilter.create({ backend: "webgpu" });
```

Both share the same OPFS cache for the 770 MB model вҖ” switching is a ~1.1-second warmup, not a full re-download.

## Memory footprint

| Backend | Resident memory during inference |
|---|---|
| `webgpu` | ~30 MB JS heap + ~800 MB GPU buffers |
| `wasm` | ~800 MB WASM linear memory |
| `auto` (transformers.js path) | ~870 MB JS heap + ~1 GB GPU |

`webgpu` is the most memory-efficient since weights live in GPU buffers and JS holds only kernel-dispatch state.
