All numbers measured on an M3 Pro MacBook, Chromium 147 with `--enable-unsafe-webgpu --use-angle=metal`. Median of 5 samples after 2 warmup iterations. Reproduce with the test in `tests/browser/bench.spec.ts`.

These numbers will look different on your hardware. They're a snapshot, not a guarantee.

## Per-forward latency

| Input length | textsift (WebGPU) | textsift (WASM ST) | tjs (WebGPU default) |
|---|---:|---:|---:|
| ~12 tokens | 7 ms | 66 ms | 29 ms |
| ~25 tokens | 12 ms | 158 ms | 39 ms |
| ~80 tokens | 25 ms | 342 ms | 57 ms |

Both WebGPU paths are GPU-bound at this model size; the difference reflects the kernels (textsift's hand-tuned int4 matmul / banded attention vs ORT-Web's int4 implementation via WebGPU contrib ops). The WASM single-thread number jumps to ~94 ms at 32 tokens with multi-threading enabled (COOP/COEP headers + 6 worker threads).

## Cold start

This is a default-configuration comparison, not an architectural ceiling.

| | First visit | Second visit |
|---|---:|---:|
| tjs (Cache API default) | 13.3 s | 11.2 s |
| textsift WASM (OPFS default) | 14.1 s | 1.8 s |
| textsift WebGPU (OPFS default) | 14.1 s | 1.1 s |

First visit is download-bound (~770 MB from HuggingFace CDN); the time goes to bytes, not code.

Second visit: textsift writes the 770 MB model to OPFS by default. transformers.js's default cache is the Cache API, which silently rejects payloads this large — so its uncached path runs every visit. A transformers.js user who plugs in a custom OPFS-backed cache can match textsift's second-visit warmup. The factual claim is **"textsift caches large models persistently with no configuration"**, not "textsift can cache and tjs can't".

## Warmup breakdown (cache hit)

From `tests/browser/webgpu-coldstart.spec.ts`, second warmup:

| Step | Cost |
|---|---:|
| Adapter + device request | 0.5 s (Metal driver warm-up) |
| OPFS read (770 MB model) | 0.4 s |
| ONNX parse | 0.002 s |
| GPU buffer upload | 0.4 s |
| Pipeline compile (14 WGSL shaders, parallel) | 0.002 s |
| **Total** | **1.3 s** |

## Accuracy

`openai/privacy-filter` reports F1 96% on PII-Masking-300k in OpenAI's own evaluation. textsift doesn't change the model — all three backends produce byte-identical span output on the same input (`tests/browser/smoke.spec.ts`, `webgpu-smoke.spec.ts`). The detection accuracy is the model's, not ours.

The forward-parity test (`tests/browser/webgpu-forward.spec.ts`) confirms argmax agrees on 32/32 tokens of a deterministic input, with logit RMS drift of 0.18 vs the WASM reference — fp16 rounding compounding across 8 transformer layers. Argmax (and downstream Viterbi + span output) is preserved.

## Reproduce

```sh
npm install
npm run build
npx playwright test bench.spec.ts
```

First run downloads the 770 MB model to OPFS (~15 seconds), subsequent runs are cache-hit fast.
