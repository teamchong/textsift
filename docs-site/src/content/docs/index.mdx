## Why

OpenAI released [openai/privacy-filter](https://huggingface.co/openai/privacy-filter) on 2026-04-20 — a 1.5B-parameter MoE token classifier (50M active) for PII detection, Apache 2.0. It reaches **96-97% F1 on PII-Masking-300k** and runs on a laptop.

The gap: the official SDK is Python. transformers.js runs it on WebGPU but has no CPU fallback. **`pii-wasm` is the only package that runs this model end-to-end in the browser with both WebGPU and CPU paths.**

## 30-second example

```ts
import { PrivacyFilter } from "pii-wasm";

const filter = await PrivacyFilter.create();
const result = await filter.redact(
  "Hi, my name is John Smith and my email is john@example.com.",
);

console.log(result.redactedText);
// "Hi, my name is [private_person] and my email is [private_email]."

console.log(result.spans);
// [
//   { label: "private_person", start: 15, end: 25, text: "John Smith", ... },
//   { label: "private_email",  start: 43, end: 59, text: "john@example.com", ... },
// ]
```

## How it compares

| | tjs (WebGPU) | **pii-wasm (WebGPU)** | pii-wasm (WASM) |
|---|---:|---:|---:|
| Forward — short | 29 ms | **7 ms** | 66 ms |
| Forward — medium | 39 ms | **12 ms** | 158 ms |
| Forward — long | 57 ms | **25 ms** | 342 ms |
| Second-visit warmup | 11.2 s | **1.1 s** | 1.8 s |

*M-series MacBook, Chromium 147, median of 5 runs after 2 warmups.*

For **CPU-only environments** (corporate Chrome without WebGPU, Firefox with `dom.webgpu.enabled=false`, server-side Node), pii-wasm is the only working path — transformers.js fails session creation on this model's WASM device due to missing ORT-Web kernels (`GatherBlockQuantized`, `MatMulNBits`).
